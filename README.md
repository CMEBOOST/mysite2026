# 📨 คู่มือสรุปสาระสำคัญของ HTTP Request

HTTP Request คือข้อความคำขอที่ฝั่งผู้ใช้ (Client เช่น เว็บเบราว์เซอร์ หรือแอปพลิเคชัน) ส่งไปหาเว็บเซิร์ฟเวอร์ เพื่อร้องขอข้อมูลหรือสั่งให้เซิร์ฟเวอร์ดำเนินการอย่างใดอย่างหนึ่ง

---

## 🏗️ โครงสร้างของ HTTP Request (Request Structure)

ข้อความ HTTP Request จะถูกจัดส่งโดยแบ่งออกเป็น 4 ส่วนหลักตามลำดับดังนี้:

```text
[Request Line]   -> GET /v1/users HTTP/1.1
[HTTP Headers]   -> Host: ://example.com
                    Authorization: Bearer xyz123
                    Accept: application/json
[Empty Line]     -> (บรรทัดว่าง 1 บรรทัด เพื่อแยกส่วนหัวกับเนื้อหา)
[Request Body]   -> { "name": "John Doe", "email": "john@example.com" }
```

### 1. Request Line (บรรทัดคำขอ)
บรรทัดแรกสุดที่บอกคำสั่งพื้นฐาน ประกอบด้วย 3 ส่วน:
* **HTTP Method** - คำสั่งที่ต้องการทำ (เช่น GET, POST)
* **Path / URL** - เส้นทางทรัพยากรบนเซิร์ฟเวอร์ (เช่น `/index.html`, `/api/products`)
* **HTTP Version** - เวอร์ชันของโปรโตคอล (เช่น `HTTP/1.1`, `HTTP/2`)

### 2. HTTP Headers (ข้อมูลส่วนหัว)
ข้อมูลเสริมในรูปแบบ `Key: Value` เพื่อให้รายละเอียดเพิ่มเติมเกี่ยวกับคำขอ เช่น:
* `Host:` โดเมนของเซิร์ฟเวอร์ปลายทาง (เช่น `://example.com`)
* `Content-Type:` ประเภทของข้อมูลที่แนบมาใน Body (เช่น `application/json`, `multipart/form-data`)
* `Authorization:` โทเคนหรือรหัสผ่านที่ใช้ยืนยันตัวตนเพื่อเข้าถึงระบบ
* `User-Agent:` ข้อมูลเกี่ยวกับโปรแกรมหรือเบราว์เซอร์ที่ใช้ส่งคำขอ

### 3. Empty Line (บรรทัดว่าง)
* บรรทัดว่างเปล่าจำนวน 1 บรรทัด ทำหน้าที่เป็นตัวคั่นตามมาตรฐาน เพื่อบอกเซิร์ฟเวอร์ว่าส่วน Headers สิ้นสุดแล้ว และถัดไปจะเป็นส่วนของ Body

### 4. Request Body (เนื้อหาข้อมูล)
* ข้อมูลหรือ Payload ที่ต้องการส่งไปให้เซิร์ฟเวอร์ประมวลผล 
* มักใช้กับ Method `POST`, `PUT`, `PATCH` (เช่น ข้อมูลฟอร์มสมัครสมาชิก, ไฟล์รูปภาพ, ข้อมูล JSON)
* สำหรับ Method `GET` และ `DELETE` โดยปกติจะไม่มีการส่งเนื้อหาในส่วนนี้

## 🛠️ HTTP Methods ที่นิยมใช้งาน (CRUD Mapping)

HTTP Method คือตัวกำหนดประเภทการกระทำ โดยส่วนใหญ่นิยมใช้จับคู่กับการจัดการข้อมูลแบบ CRUD ดังนี้:

| HTTP Method | การทำงาน (Action) | การจับคู่ระบบ CRUD | มี Request Body? |
| :--- | :--- | :--- | :--- |
| **`GET`** | ขอรับ/ดึงข้อมูลจากเซิร์ฟเวอร์ | **R**ead (อ่าน) | ❌ ไม่มี |
| **`POST`** | ส่งข้อมูลไปสร้างสิ่งใหม่ | **C**reate (สร้าง) |  มี |
| **`PUT`** | ส่งข้อมูลไปแก้ไขแทนที่ตัวเดิมทั้งหมด | **U**pdate (อัปเดต ทั้งชุด) |  มี |
| **`PATCH`** | ส่งข้อมูลไปแก้ไขบางส่วน | **U**pdate (อัปเดต บางจุด) |  มี |
| **`DELETE`** | สั่งลบข้อมูลที่ระบุ | **D**elete (ลบ) | ❌ ไม่มี (ส่วนใหญ่) |

# 🚀 คู่มือสรุปสาระสำคัญของ HTTP Status Codes

รหัสสถานะ HTTP (HTTP Status Code) คือรหัสตัวเลข 3 หลักที่เว็บเซิร์ฟเวอร์ส่งกลับมาเพื่อตอบสนองคำขอ (Request) จากเบราว์เซอร์หรือไคลเอนต์ เพื่อบอกสถานะการทำงานว่าเป็นอย่างไร

---

## 📂 ประเภทของรหัสสถานะ (Status Categories)

| ช่วงรหัส | ประเภท | ความหมาย |
| :--- | :--- | :--- |
| **1xx** | Informational | เซิร์ฟเวอร์ได้รับคำขอแล้ว กำลังดำเนินการ |
| **2xx** | Success | คำขอสำเร็จ ดำเนินการและตอบกลับข้อมูลเรียบร้อย |
| **3xx** | Redirection | ต้องดำเนินการเปลี่ยนเส้นทางเพิ่มเติม |
| **4xx** | Client Error | เกิดข้อผิดพลาดจากฝั่งผู้ใช้งาน (เช่น URL ผิด, ไม่มีสิทธิ์) |
| **5xx** | Server Error | เกิดข้อผิดพลาดจากฝั่งเซิร์ฟเวอร์ (ไม่สามารถประมวลผลได้) |

---

## 📌 รหัสสถานะที่พบบ่อย (Common HTTP Status Codes)

### 🟢 2xx: Success (สำเร็จ)
* **`200 OK`** - คำขอสำเร็จและเซิร์ฟเวอร์ส่งข้อมูลกลับมาปกติ
* **`201 Created`** - คำขอสำเร็จ และมีการสร้างทรัพยากรใหม่บนเซิร์ฟเวอร์แล้ว

### 🟡 3xx: Redirection (การเปลี่ยนเส้นทาง)
* **`301 Moved Permanently`** - ย้ายหน้าเว็บไปยัง URL ใหม่แบบถาวร
* **`302 Found`** - ย้ายหน้าเว็บไปยัง URL ใหม่แบบชั่วคราว
* **`304 Not Modified`** - ข้อมูลเดิมยังใช้งานได้ (ดึงจาก Cache) ไม่ต้องโหลดใหม่

### 🔴 4xx: Client Error (ข้อผิดพลาดฝั่งผู้ใช้)
* **`400 Bad Request`** - คำขอไม่ถูกต้องตามโครงสร้างไวยากรณ์
* **`401 Unauthorized`** - ปฏิเสธการเข้าถึงเนื่องจากยังไม่ได้เข้าสู่ระบบ (ต้องกดยืนยันตัวตน)
* **`403 Forbidden`** - เซิร์ฟเวอร์เข้าใจคำขอ แต่ผู้ใช้ไม่มีสิทธิ์เข้าถึงทรัพยากรนั้น
* **`404 Not Found`** - ไม่พบหน้าเว็บหรือทรัพยากรที่เรียกหา
* **`405 Method Not Allowed`** - มีการส่ง Method (เช่น GET/POST) ที่ระบบไม่รองรับ

### 🛑 5xx: Server Error (ข้อผิดพลาดฝั่งเซิร์ฟเวอร์)
* **`500 Internal Server Error`** - เกิดข้อผิดพลาดทั่วไปภายในระบบเซิร์ฟเวอร์
* **`502 Bad Gateway`** - เซิร์ฟเวอร์ทำหน้าที่เป็น Gateway และได้รับคำตอบที่ผิดพลาดจากเซิร์ฟเวอร์ต้นทาง
* **`503 Service Unavailable`** - เซิร์ฟเวอร์ปิดปรับปรุงชั่วคราว หรือใช้งานหนาแน่นเกินไป
* **`504 Gateway Timeout`** - เซิร์ฟเวอร์ที่เป็น Gateway รอการตอบกลับนานจนหมดเวลา
